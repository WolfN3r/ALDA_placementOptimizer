# ALDA Placement Viewer

A KLayout-style desktop viewer for ALDA placement JSON files.

---

## Requirements

```
pip install PyQt6
```

---

## Starting the app

From anywhere in the project:

```bash
# Open empty viewer
python WorkDir/viewer/main.py

# Open with a file immediately
python WorkDir/viewer/main.py WorkDir/json_files/s42_n10_py001_v01.json
```

The path you pass is resolved relative to your CWD, then relative to `WorkDir/`, then relative to the project root — so the above works regardless of which directory you run from.

---

## File format

The viewer loads `s<seed>_n<blocks>_py<stage>_v<version>.json` files generated by the ALDA pipeline.  
Both direct-dict format (from `main.py`) and n8n list-wrapped format (from `testFiles/`) are supported.

## Interface

```
┌──────────────┬─────────────────────────────┬──────────────────┐
│ Layers panel │   Block tabs (one per block) │ Block info panel │
│              │                              │                  │
│ Toggle which │   Canvas: shows the block    │ Block metadata   │
│ layer types  │   as a bbox rectangle with   │ Variant picker   │
│ are visible  │   net stubs going outward    │ Pin → net list   │
│              │                              │ Grid settings    │
└──────────────┴─────────────────────────────┴──────────────────┘
│ Mouse coordinates (µm)                          Mode | Blocks  │
```

### Navigation

| Action | How |
|--------|-----|
| Zoom in / out | Scroll wheel |
| Pan | Middle-click drag or right-click drag |
| Fit to view | `F` key or **Fit [F]** button |
| Open file | **File → Open JSON** or **Open** button |

### Tabs

Each block in the JSON gets its own tab (`B0 nmos1v`, `B1 nmos1v`, …).  
Click a tab to inspect that block.

### Block info panel (right)

- **Variant picker** — switch between layout variants of the selected block; the canvas updates and re-fits automatically.
- **Pins** — lists each pin and the net it connects to, plus which other blocks share that net.
- **Grid** — set the main step and small step (in µm); grid starts at the scene origin `(0, 0)`.

### Layers panel (left)

Toggle visibility of:

| Layer | What it shows |
|-------|---------------|
| Block outline (Annotation) | The `main_bbox` rectangle, colored by device type |
| Power nets (VDD/VSS) | Net stubs for VDD (red) and VSS (blue) |
| Signal nets | Net stubs for signal connections (green) |
| Labels | Pin names and net labels on stubs |
| Symmetry pairs | (future) symmetry group overlays |

### Net stubs

Each pin draws a colored line from the block center outward to the edge and beyond.  
At the stub end: `P0 → net_id` with the connected block IDs in parentheses, e.g. `(B1, B4)`.

Power pins (at the top or bottom bbox edge) exit vertically; signal pins alternate left/right.

---

## Placement mode

When the optimizer adds a `"placed"` key to a block (containing `x`, `y`, `rotation_deg`, `variant_index`), the viewer automatically switches from **Browser** mode to **Placed** mode and draws blocks at their optimizer-output coordinates.
