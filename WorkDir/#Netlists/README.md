# ALIGN Netlist Collection — Reference Guide

This folder contains 47 analog circuit netlists in ALIGN-compatible SPICE format.
All files are generated by `convert_netlists.py` from two source sets:

| Series | Pattern | Count | Origin |
|--------|---------|-------|--------|
| **a-series** | `a<XY>_<name>.sp` | 41 | ALIGN FinFET example circuits, converted to CMOS style |
| **m-series** | `m<XY>_<name>.sp` | 6 | MAGICAL real-silicon designs (TSMC 40 nm), converted from Spectre/HSPICE |

---

## 1. File Structure

Every netlist follows this layout (top to bottom):

```
* ============================================================
* Circuit : <name>
* Source  : <origin>
* File    : <relative source path>
* ============================================================
* Stats:
*   Subcircuits  : N
*   Devices (M)  : N
*   Device types : <space-separated list>
*   Passives     : N resistors, N capacitors
* Note: ...
* ============================================================

* --- DEVICE PARAMETERS ---          ← present only if transistors have literal sizes
.param M1_L=120n  M1_W=4u  M1_Nf=2  M1_M=1
.param M2_L=120n  M2_W=8u  M2_M=1   M2_Nf=4
...

* --- CIRCUIT DEFINITION ---         ← separator when param block exists
.subckt <top_cell> <port1> <port2> ... <VDDA> <VSSA>
    ...device lines...
.ends <top_cell>
```

The netlist may contain multiple `.subckt` blocks when the design uses helper
cells or multiple stages. All blocks are defined in dependency order (leaves
first, root last).

---

## 2. MOSFET Device Types

Each MOSFET line names one of six canonical model types:

| Model name  | Channel | Threshold voltage |
|-------------|---------|------------------|
| `nmos_rvt`  | N       | Regular (standard) |
| `pmos_rvt`  | P       | Regular (standard) |
| `nmos_lvt`  | N       | Low — faster, higher leakage |
| `pmos_lvt`  | P       | Low — faster, higher leakage |
| `nmos_hvt`  | N       | High — slower, lower leakage |
| `pmos_hvt`  | P       | High — slower, lower leakage |

`nmos_hvt` / `pmos_hvt` appear only in m-series files (MAGICAL circuits).
The a-series uses only `nmos_rvt`, `pmos_rvt`, `nmos_lvt`, `pmos_lvt`.

## 3. MOSFET Line Format

```
<instance>  <drain>  <gate>  <source>  <bulk>  <model>  W=<w>  L=<l>  Nf=<nf>  M=<m>
```

Pin order is always **D G S B** — Drain, Gate, Source, Bulk (body).

| Parameter | Meaning | Typical unit |
|-----------|---------|-------------|
| `W`  | Channel width (total or per finger) | µm (e.g. `4u`) or nm (e.g. `480n`) |
| `L`  | Channel length | nm or µm (e.g. `120n`, `0.014u`) |
| `Nf` | Number of gate fingers | integer |
| `M`  | Multiplicity (parallel copies of the cell) | integer |

### Bulk (body) connection convention

The bulk pin determines the well connection:

- **NMOS**: bulk tied to the **most negative supply** (`VSSA`) — keeps the p-substrate reverse-biased.
- **PMOS**: bulk tied to the **most positive supply** (`VDDA`) — keeps the n-well reverse-biased.

```spice
* Typical NMOS — bulk to VSSA
MN1  drain  gate  source  VSSA  nmos_rvt  W={MN1_W}  L={MN1_L}  Nf={MN1_Nf}  M={MN1_M}

* Typical PMOS — bulk to VDDA
MP1  drain  gate  source  VDDA  pmos_rvt  W={MP1_W}  L={MP1_L}  Nf={MP1_Nf}  M={MP1_M}
```

When the bulk is connected to a non-global net (body-effect bias or substrate
isolation), you will see a local net name instead.

---

## 4. Power Rail Convention

| Net name | Role |
|----------|------|
| `VDDA`   | Analog positive supply |
| `VSSA`   | Analog ground / negative supply |
| `VDDD`   | Digital positive supply (mixed-signal designs) |
| `VSSD`   | Digital ground (mixed-signal designs) |

Power rails are always listed as the **last ports** in `.subckt` declarations,
e.g. `.subckt five_transistor_ota vinp vinn vout VDDA VSSA`.

---

## 5. Parameter Extraction

Transistor dimensions are never hard-coded inside the netlist body. Instead
they are lifted into a `.param` block at the top of the file and referenced via
`{param_name}` braces:

```spice
.param MN1_L=120n  MN1_W=4u  MN1_Nf=2  MN1_M=1

* later in the subckt body:
MN1  vout  vin  VSSA  VSSA  nmos_lvt  W={MN1_W}  L={MN1_L}  Nf={MN1_Nf}  M={MN1_M}
```

Naming scheme for parameter keys:

| Context | Key format | Example |
|---------|-----------|---------|
| Single-subckt file | `<instance>_<PARAM>` | `MN1_W` |
| Multi-subckt file  | `<subckt>_<instance>_<PARAM>` | `nfet2x_MN0_L` |

The `PARAM` suffix is always one of `W`, `L`, `Nf`, `M`.

Non-literal values (expressions, references) are kept inline and are **not**
extracted, e.g. `Nf=p1` when `p1` is a subcircuit parameter.

## 6. Passive Elements

### Standard SPICE passives (a-series)

```spice
R<name>  <node1>  <node2>  <value>        * resistor with scalar value
C<name>  <node1>  <node2>  <value>        * capacitor with scalar value
```

Example: `R0  vbn1  n1  20K`   `C0  vin  vbn1  48f`

Values use standard SPICE suffixes: `f`=femto, `p`=pico, `n`=nano, `u`=micro,
`m`=milli, `K`=kilo, `M`=mega.

### PDK-cell passives (m-series, MAGICAL)

MAGICAL circuits use metal-oxide-metal capacitors and polysilicon resistors
defined in the foundry PDK. These are kept as subcircuit instances (x-prefix):

```spice
* MOM capacitor — 2-terminal
xC1  <node1>  <node2>  cfmom_2t  nr=210  lr=13.0u  w=70n  ...

* MOM capacitor — 3-terminal (with substrate node)
xC7  <node1>  <node2>  <substrate>  cfmom  nr=32  lr=2.7u  ...

* Polysilicon resistor — 3-terminal (with substrate node)
xR11  <node1>  <node2>  <substrate>  rppolywo  lr=3.6u  wr=400n  series=6  ...
```

The geometric parameters (`nr`, `lr`, `wr`, `series`, …) are PDK-specific and
define the physical layout of the device. ALIGN uses the mockPDK definitions in
`MAGICALexamples/mockPDK/` to resolve these cells.

---

## 7. Subcircuit Instances

Non-primitive instantiations (calls to user-defined or PDK subckts) use the
`x` prefix:

```spice
x<name>  <node1>  <node2>  ...  <subckt_type>  [key=value ...]
```

Example: `xI0  vout  vbn1  VSSA  VSSA  nfet2x  p1=4`

The subcircuit type is always the **last positional token** before any
`key=value` parameters.

## 8. Annotated Example

```spice
* ============================================================
* Circuit : five_transistor_ota
* ============================================================

* --- DEVICE PARAMETERS ---
.param  MN1_L=120n  MN1_W=8u  MN1_Nf=4  MN1_M=1   * diff-pair left
.param  MN2_L=120n  MN2_W=8u  MN2_Nf=4  MN2_M=1   * diff-pair right
.param  MP1_L=240n  MP1_W=16u MP1_Nf=4  MP1_M=1   * load mirror left
.param  MP2_L=240n  MP2_W=16u MP2_Nf=4  MP2_M=1   * load mirror right
.param  MNT_L=240n  MNT_W=4u  MNT_Nf=2  MNT_M=1   * tail current source

* --- CIRCUIT DEFINITION ---
.subckt five_transistor_ota  vinp  vinn  vout  VDDA  VSSA
*
*  D         G     S      B      MODEL
   MN1  outn  vinp  tail  VSSA  nmos_rvt  W={MN1_W}  L={MN1_L}  Nf={MN1_Nf}  M={MN1_M}
   MN2  vout  vinn  tail  VSSA  nmos_rvt  W={MN2_W}  L={MN2_L}  Nf={MN2_Nf}  M={MN2_M}
   MP1  outn  outn  VDDA  VDDA  pmos_rvt  W={MP1_W}  L={MP1_L}  Nf={MP1_Nf}  M={MP1_M}
   MP2  vout  outn  VDDA  VDDA  pmos_rvt  W={MP2_W}  L={MP2_L}  Nf={MP2_Nf}  M={MP2_M}
   MNT  tail  vbias VSSA  VSSA  nmos_rvt  W={MNT_W}  L={MNT_L}  Nf={MNT_Nf}  M={MNT_M}
.ends five_transistor_ota
```

Reading guide:
- `MN1` drives its **drain** to `outn`, is controlled by `vinp` at its **gate**, sources current from `tail`, and has its **bulk** tied to `VSSA` (ground).
- `MP1` is a diode-connected PMOS load: drain = gate = `outn`, source and bulk to `VDDA`.
- `MNT` is the tail current source: drain feeds `tail`, gate biased by `vbias`, source and bulk to `VSSA`.

## 9. How the Converter Works

`convert_netlists.py` in this folder generates all netlists. Running it:

```
python examples/#Netlists/convert_netlists.py
```

What it does:

1. **Reads** each source `.sp` file (a-series from `examples/<name>/`, m-series from `MAGICALexamples/`)
2. **Normalises format** — joins continuation lines, converts `//` to `*` comments, normalises Spectre/HSPICE syntax to plain SPICE (for m-series)
3. **Remaps device types** — e.g. `nch_lvt_mac` → `nmos_lvt`, `pch` → `pmos_rvt`
4. **Remaps power rails** per circuit — e.g. `vps`/`vgnd` → `VDDA`/`VSSA`
5. **Extracts literal sizes** into a `.param` block and replaces them with `{param_name}` references
6. **Renames parameters** — `nfin`→`W`, `l`→`L`, `nf`→`Nf`, `m`/`multi`→`M`
7. **Writes** the output file (`a<XY>_` or `m<XY>_` prefix)
